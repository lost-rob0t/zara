* Goals
This page covers installation and configuration paths for Zarathushtra.
The project prefers Nix for repeatable environments and does not offer
support for ad-hoc setups.

* Nix (Recommended)
** Requirements
- Nix with flakes enabled
- Network access for dependencies

** Development shell
#+begin_src sh
nix develop
#+end_src

** Validation
#+begin_src sh
nix develop -c pytest
nix develop -c scripts/test-command-routing.sh
nix develop -c scripts/test-replies.sh
nix develop -c scripts/test-prolog-config.sh
nix develop -c scripts/test-dictation-lifecycle.sh
nix develop -c scripts/test-file-tools.sh
nix develop -c scripts/test-llm-clients.sh
nix develop -c scripts/test-process-safety.sh
nix develop -c scripts/test-prolog-engine.sh
nix develop -c scripts/test-timers.sh
#+end_src

The command-routing gate verifies that unsupported Prolog handlers do not report
a successful execution. The reply gate verifies that the noncommittal
acknowledgement precedes execution and that structured success and failure
results reach both notification and TTS sinks.

** Build all packages
#+begin_src sh
nix build
#+end_src

** Run entrypoints
#+begin_src sh
nix run
nix run .#zara-wake
nix run .#zara-console
nix run .#zara-dictate
#+end_src

Dictation stop phrases use the canonical table in =zara.dictate=. Configure
phrases with =[dictate].stop_phrases=, =--stop-phrases=,
=ZARA_STOP_PHRASE=, or =ZARA_STOP_PHRASES=. Matching ignores case, spacing, and
punctuation. Parallel transcription remains enabled, while text is committed in
capture order.

* Python (Unsupported)
This repo ships a requirements file for reference only. It is not supported.
Use it only if you fully understand the dependency set and how to provide
SWI-Prolog, audio backends, and model assets outside Nix.

#+begin_src sh
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
#+end_src

* Configuration

Zara creates =~/.config/zarathushtra/config.toml= by default, or uses
=$XDG_CONFIG_HOME/zarathushtra/config.toml= when =XDG_CONFIG_HOME= is set.
Invalid TOML stops startup with the configuration path and parser diagnostic so
user overrides are never silently discarded. The canonical Qwen TTS provider is
=qwen3=; existing configurations using =qwen= remain supported as an alias.
All providers return declared WAV or MP3 data through a structured synthesis
result. =[tts].connect_timeout=, =read_timeout=, and =total_timeout= bound remote
work. Starting a new utterance cancels and replaces the active playback.

The Python and Prolog LLM clients serialize Anthropic, OpenAI, and Ollama
requests according to each provider's protocol. Configure
=[llm].connect_timeout=, =read_timeout=, =total_timeout=, =max_retries=, and
=history_limit= to bound remote work and retained conversation context. The
Python client reuses one HTTP session until shutdown; Prolog uses bounded
keep-alive connections. Provider failures remain typed until the user-facing
boundary, so an HTTP or protocol error is never stored as assistant text.
Anthropic and OpenAI credentials come from =ANTHROPIC_API_KEY= and
=OPENAI_API_KEY=, or the existing matching =[llm]= key settings. Leave
=[llm].endpoint= empty to use the selected provider's default endpoint; set it
only for an explicit compatible proxy or local service.

Python runtime settings use =config.toml=. Prolog facts remain separate in
=~/.config/zarathushtra/config.pl=. That Prolog file accepts only the generated
fact forms; they are validated, installed into =kb_config= or =kb_intents=, and
replace prior user overrides on reload.

Dictation process state is determined from the configured PID file and live
process. =ZARA_DICTATION_PIDFILE= and =ZARA_DICTATION_LOGFILE= can override the
default paths for isolated environments.

Prolog application and dictation commands should be configured as executable
and argument lists, for example =app_mapping(browser, ["xdg-open",
"https://example.com"])= and =dictation_command(["zara-dictate", "small"])=.
Legacy whitespace-only strings remain accepted when every token uses the
constrained command character set. Shell syntax is rejected, and runtime
launches never interpolate command text through a shell.

Timer parsing, unit conversion, scheduling, and completion notifications are
implemented in Prolog. Timer commands use the canonical shape
=[Seconds, Name]=; alarms remain explicitly unsupported.

Timer and alarm sounds are separate Prolog settings in =config.pl=:
=timer_sound/1= and =alarm_sound/1=. Both accept a readable audio-file path;
the bundled defaults live in =assets/sounds/= and play through =mpv=.

Intent precedence is state-aware in Prolog through =resolve/4=. Conversation,
dictation, and passive controls are distinct; missing command slots return a
typed =pending(...)= result for the thin Python boundary to present.

=PrologEngine= raises =PrologStartupError= when its main program cannot be
consulted and =PrologQueryError= for runtime or syntax failures. A logical
no-solution remains =None=, so it cannot be confused with an engine failure.
PySWIP uses a process-wide SWI-Prolog runtime: Zara serializes every consult and
query across all =PrologEngine= instances. Lazy query iterators retain that
runtime lock until exhausted or explicitly closed. User-derived strings, atoms,
and lists pass through the engine's constrained term serializers before a goal
is submitted.

The Prolog command loop emits a short =acknowledgement= event before resolution
or execution, followed by one =command_result(Status, Intent, Args, Detail)=
event after execution. Final reply text uses the resolved arguments, and failed
handlers produce failure text instead of a completion claim. Concise replies are
the default. Set =ZARA_RICH_REPLIES=1= to include optional philosophical phrases.
Set =ZARA_REPLY_TTS_COMMAND= to an executable name when Prolog replies should
also be sent to a TTS command; reply sink failures never determine command
success.

Agent history validation preserves complete contiguous tool-result groups in
the originating AI tool-call order. Orphaned, duplicate, unknown, and
cross-turn tool results are discarded before provider serialization.

Agent file tools are disabled by default with =tools.file_tools = false=. To
enable them, set that value to =true= and configure =file_tools.readable_roots=,
=file_tools.writable_roots=, and =file_tools.max_bytes=. Relative roots resolve
from the repository root. Paths outside those roots, symlinks, and special files
are rejected; writes require explicit overwrite permission and are atomic.
** Config file
Zarathushtra reads configuration from:
- ~/.config/zarathushtra/config.toml

The file is created on first run. You can also provide a custom path when
instantiating the config manager.

** Key sections
- [agent]: LLM settings, system prompt, and timeouts
- [prolog]: main file and load-on-startup toggle
- [tools]: tool enable/disable flags
- [modules]: plugin search paths and autoload list
- [wake], [stt], [tts], [dictate], [memory], [noaa]
- Memory falls back to process-local storage when ChromaDB or its embedding backend fails. The manager reports =disabled=, =persistent=, or =local_fallback= through =get_health()=.
- =[memory].summary_max_chars= bounds stored session summaries and transcripts. Raw session capture is stored as =transcript=; only caller-supplied summaries are stored as =summary=.

** Wake audio sample rate
- =wake.sample_rate= sets the target rate for transcription.
- If the input device does not support that rate, Zarathushtra falls back to the
  device default and resamples audio to the target.
- =wake.audio_queue_chunks= bounds the real-time callback buffer. When full, the
  oldest chunk is dropped so current audio wins.
- =wake.first_speech_timeout= returns a wake with no speech to passive mode.
- =wake.silence_duration= ends an utterance after speech followed by silence.
- =wake.max_utterance_duration= caps continuous speech or noise. Conversation
  mode separately uses =agent.conversation_timeout= as its inactivity deadline.

** Example: set a custom system prompt
#+begin_src toml
[agent]
system_prompt = "/home/you/.config/zarathushtra/system_prompt.txt"
#+end_src
