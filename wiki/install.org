* Goals
This page covers installation and configuration paths for Zarathushtra.
The project prefers Nix for repeatable environments and does not offer
support for ad-hoc setups.

* Nix (Recommended)
** Requirements
- Nix with flakes enabled
- Network access for dependencies

** Development shell
#+begin_src sh
nix develop
#+end_src

** Validation
#+begin_src sh
nix develop -c pytest
nix develop -c scripts/test-command-routing.sh
nix develop -c scripts/test-prolog-config.sh
nix develop -c scripts/test-dictation-lifecycle.sh
nix develop -c scripts/test-timers.sh
#+end_src

The command-routing gate verifies that unsupported or failed Prolog handlers
fall back without reporting a successful execution.

** Build all packages
#+begin_src sh
nix build
#+end_src

** Run entrypoints
#+begin_src sh
nix run
nix run .#zara-wake
nix run .#zara-console
nix run .#zara-dictate
#+end_src

* Python (Unsupported)
This repo ships a requirements file for reference only. It is not supported.
Use it only if you fully understand the dependency set and how to provide
SWI-Prolog, audio backends, and model assets outside Nix.

#+begin_src sh
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
#+end_src

* Configuration

Zara creates =~/.config/zarathushtra/config.toml= by default, or uses
=$XDG_CONFIG_HOME/zarathushtra/config.toml= when =XDG_CONFIG_HOME= is set.
Invalid TOML stops startup with the configuration path and parser diagnostic so
user overrides are never silently discarded. The canonical Qwen TTS provider is
=qwen3=; existing configurations using =qwen= remain supported as an alias.
All providers return declared WAV or MP3 data through a structured synthesis
result. =[tts].connect_timeout=, =read_timeout=, and =total_timeout= bound remote
work. Starting a new utterance cancels and replaces the active playback.

Python runtime settings use =config.toml=. Prolog facts remain separate in
=~/.config/zarathushtra/config.pl=. That Prolog file accepts only the generated
fact forms; they are validated, installed into =kb_config= or =kb_intents=, and
replace prior user overrides on reload.

Dictation process state is determined from the configured PID file and live
process. =ZARA_DICTATION_PIDFILE= and =ZARA_DICTATION_LOGFILE= can override the
default paths for isolated environments.

Timer parsing, unit conversion, scheduling, and completion notifications are
implemented in Prolog. Timer commands use the canonical shape
=[Seconds, Name]=; alarms remain explicitly unsupported.

Timer and alarm sounds are separate Prolog settings in =config.pl=:
=timer_sound/1= and =alarm_sound/1=. Both accept a readable audio-file path;
the bundled defaults live in =assets/sounds/= and play through =mpv=.

Intent precedence is state-aware in Prolog through =resolve/4=. Conversation,
dictation, and passive controls are distinct; missing command slots return a
typed =pending(...)= result for the thin Python boundary to present.

Agent history validation preserves complete contiguous tool-result groups in
the originating AI tool-call order. Orphaned, duplicate, unknown, and
cross-turn tool results are discarded before provider serialization.
** Config file
Zarathushtra reads configuration from:
- ~/.config/zarathushtra/config.toml

The file is created on first run. You can also provide a custom path when
instantiating the config manager.

** Key sections
- [agent]: LLM settings, system prompt, and timeouts
- [prolog]: main file and load-on-startup toggle
- [tools]: tool enable/disable flags
- [modules]: plugin search paths and autoload list
- [wake], [stt], [tts], [dictate], [memory], [noaa]
- Memory falls back to process-local storage when ChromaDB or its embedding backend fails. The manager reports =disabled=, =persistent=, or =local_fallback= through =get_health()=.
- =[memory].summary_max_chars= bounds stored session summaries and transcripts. Raw session capture is stored as =transcript=; only caller-supplied summaries are stored as =summary=.

** Wake audio sample rate
- =wake.sample_rate= sets the target rate for transcription.
- If the input device does not support that rate, Zarathushtra falls back to the
  device default and resamples audio to the target.

** Example: set a custom system prompt
#+begin_src toml
[agent]
system_prompt = "/home/you/.config/zarathushtra/system_prompt.txt"
#+end_src
